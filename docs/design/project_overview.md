# Project Overview: Tower Engine

The Tower Engine is designed as a reusable game engine, serving as the foundational spine for multiple game titles.

## Titles

*   **Damian: What Survives the Tower**: The inaugural title utilizing the Tower Engine. Its content will be structured as a dedicated content pack.
*   **Jacob's Bane**: A future title planned to run on the Tower Engine, also structured as a distinct content pack.

## Content Pack Architecture

All game-specific assets, narratives, and configurations are to be managed as data-driven content packs. This architecture ensures the engine remains title-agnostic and facilitates easier development and expansion of new games.

## Development Rules

*   No gameplay balance decisions are to be made at the engine level at this stage.
*   No assets from Blizzard Entertainment or Diablo series are to be used.
*   The engine must not contain any Damian-specific hardcoded logic or assets.
*   All content must be data-driven, allowing for easy modification and extension without engine code changes.
*   Multiplayer capabilities are reserved for future implementation and are not to be included in this foundational phase.
